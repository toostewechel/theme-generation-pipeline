# Primitives-only color engine — design

**Date:** 2026-06-27
**Status:** Approved (design)

## Summary

Make the color engine a one-way **input generator** for Figma. It generates and
exports only the **primitive** color ramps. The mapping from primitives to
**semantic** color tokens now lives in Figma, where branding/theme expansion is
low-effort. Figma is the source of truth for everything downstream of primitives;
`src/tokens/` becomes a Figma-managed snapshot that the local CSS build consumes.

## Workflow this enables

```
theme.config.ts → engine → primitives-color.mode-1 (seed)
        │ paste into Figma (overwrites primitive variables)
        ▼
   Figma: expand primitives + branding, map semantics
        │ export from Figma
        ▼
   paste back into src/tokens/ → build:tokens → dist/css
```

The engine is run to **seed** fresh primitives. Between seeds, Figma owns the
token set. Re-running `build:theme` re-seeds from scratch.

## Scope

In scope:
- Engine emits primitives only — both the Figma clipboard export and the disk write.
- Semantic generation logic is retained but unwired from file/bundle output (the
  live preview still uses it in-memory to validate ramps in context).
- Existing semantic + static-prism token files stay on disk as Figma-owned static
  files; the local CSS build (`build:tokens`) keeps consuming them unchanged.

Out of scope:
- Any change to `build:tokens`, `manifest.json`, the preview, or the CSS output.
- A feature flag / CLI opt-in to re-enable semantic export (chose the hard split).
- Importing tokens from Figma back into `src/tokens/` is a manual paste — no tooling.

## Decisions (locked)

1. **Hard split**, not a flag. "Engine emits primitives; Figma owns semantics" is a
   structural fact. `buildSemanticDtcg` / `resolveSemantics` remain exported so the
   preview keeps working and re-enabling later is a one-line re-wire.
2. **Stop writing semantic files**, but keep them on disk as Figma-owned. The engine
   no longer regenerates `color.light/dark.tokens.json`.
3. **Keep semantic files for the local CSS build.** `manifest.json` and
   `buildTokens.ts` are unchanged; the local CSS build runs on the Figma snapshot.
4. **Remove `STATIC_PRISM` from the engine entirely.** Prism colors are added in
   Figma. The physical `primitives-color.static.tokens.json` stays in `src/tokens/`
   (Figma-owned; `color.light/dark` alias `color-prism-*`, so `build:tokens` needs it).
5. **Accept the clobber caveat.** The engine owns `primitives-color.mode-1.tokens.json`
   as a seed; re-running `build:theme` overwrites it. This is non-destructive in
   practice because Figma is the source of truth between seeds.

## Round-trip integrity

The engine's primitive token names are deterministic (`rampNamePrefix` × steps), so
names are stable across regenerations. Figma's semantic variables are aliases onto
those primitive names: pasting a fresh primitives export overwrites primitive
*values*, and Figma's semantic mappings pick them up automatically. The split is safe
precisely because primitive names don't churn.

## Changes by file

### Engine

**`src/engine/figma-export.ts`**
- Remove the `STATIC_PRISM` constant and its inclusion in `buildTokenBundle`.
- `COLOR_MANIFEST` drops the `color` collection. Its `primitives-color` collection
  references only `primitives-color.mode-1.tokens.json` (not the static file — the
  engine does not export the static prism).
- `buildTokenBundle` returns a bundle with one collection and one file
  (`primitives-color.mode-1.tokens.json`).

**`src/engine/dtcg.ts`**
- `buildGeneratedFiles` returns only `primitives-color.mode-1.tokens.json`.
- `buildSemanticDtcg` stays (re-exported via `emit-dtcg.ts`; used by the preview).

**`src/engine/emit-dtcg.ts`**
- `writeGeneratedTokens` writes only `primitives-color.mode-1.tokens.json`.
- Keep re-exporting `buildSemanticDtcg`.

### Scripts

**`scripts/buildTheme.ts`**
- Update console output to list only the primitives file and adjust the "next steps"
  message to reflect the Figma round-trip.

### Token files (one-time edits)

**`src/tokens/color.light.tokens.json`, `src/tokens/color.dark.tokens.json`**
- Replace the `$description` banner `"auto-generated by build:theme — do not edit"`
  with a Figma-owned note, e.g. `"Semantic mapping owned in Figma — sync manually"`.

### Unchanged (verified)

- `src/tokens/manifest.json` — keeps the `color` collection.
- `scripts/buildTokens.ts` — still builds `:root` / `[data-color-mode='dark']` CSS
  from the semantic files.
- `tools/color-studio/src/ui/preview.ts` — still computes semantics in-memory via
  `resolveSemantics`.
- `src/tokens/primitives-color.static.tokens.json` — stays (Figma-owned).
- `src/engine/semantics.ts`, `src/engine/token-drift.ts`.

## Test changes

**`src/engine/figma-export.test.ts`**
- Bundle-filenames assertion → primitives-only (`primitives-color.mode-1.tokens.json`).
- Manifest drift check → validate only `primitives-color` matches the canonical
  manifest's primitives collection, and assert the `color` collection is intentionally
  absent from the export manifest.
- Delete "every semantic {ref} resolves to a primitive in the bundle".
- Delete "light and dark differ for at least one semantic token".
- Delete "STATIC_PRISM matches src/tokens/primitives-color.static.tokens.json".

**`src/engine/dtcg.test.ts`**
- `buildGeneratedFiles` → one file. Keep the direct `buildSemanticDtcg` tests.

**`src/engine/emit-dtcg.test.ts`**
- `writeGeneratedTokens` → one file. Keep `buildPrimitivesDtcg` / `buildSemanticDtcg`
  unit tests.

**Unchanged:** `src/engine/semantics.test.ts`, `src/engine/token-drift.test.ts`.
token-drift now also guards the hand/Figma-managed semantic files against accidental
breaking renames — a bonus, no code change.

## Verification

- `npm test` passes (with the test edits above).
- `npm run build:theme` writes only `primitives-color.mode-1.tokens.json` and logs
  accordingly.
- `npm run build:tokens` still produces `dist/css/tokens.css` with `:root` and
  `[data-color-mode='dark']` blocks (unchanged output, sourced from the Figma snapshot).
- The color-studio "Copy tokens for Figma" produces a primitives-only bundle.
- The color-studio preview/playground still renders semantic styling.
